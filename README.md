## üß≠ Introduction

**Site Reliability Engineering (SRE)** is a discipline that combines software engineering and operations to build scalable, reliable, and highly available systems.  
Originally developed at Google, SRE introduces concepts such as **SLIs, SLOs, error budgets, automation, monitoring, and eliminating toil**.

This repository aims to give beginners a structured and practical learning path into SRE.

---

## üìò Books

### **1. Site Reliability Engineering (Google SRE Book)**
- https://sre.google/sre-book/table-of-contents/
- The foundational book written by Google‚Äôs SRE team covering SLIs, SLOs, monitoring, alerting, release engineering, and team structure.

### **2. The Site Reliability Workbook**
- https://sre.google/workbook/table-of-contents/
- A companion to the SRE Book with real-world exercises, best practices, and implementation blueprints for SLIs/SLOs.

### **3. Seeking SRE** (David Blank-Edelman)
- https://www.oreilly.com/library/view/seeking-sre/9781491978856/
- Explores SRE culture, reliability at scale, and organizational implementation beyond Google.

### **4. Implementing Service Level Objectives** (Alex Hidalgo)
- https://www.oreilly.com/library/view/implementing-service-level/9781492076810/
- A practical guide to building measurable SLIs and SLOs with detailed implementation examples.

### **5. Building Secure and Reliable Systems**
- https://sre.google/books/building-secure-and-reliable-systems/
- Connects SRE and security, focusing on resilience, system hardening, and risk management.

### **6. Becoming SRE** (David Blank-Edelman, 2024)
- https://www.oreilly.com/library/view/becoming-sre/9781492090540/
- A modern introduction for new SREs, covering career progression, core competencies, and SRE fundamentals.
